задача:# ТЗ: Редкость мусора, изменение экономики дронов и фикс краша магазина

## 1. Контекст

Игра — космический idle-кликер: игрок собирает мусор (`debris`) кликом по планете
и с помощью флота дронов (`ScavengeTarget`, `DroneData`, `FleetConfig` в текущем коде).
Внизу экрана — магазин (`ShopBar`) с вкладками Planets / Fleet / Click.

Ниже — два независимых блока задач: **баг** (P0, блокирует игру) и **новая механика
экономики** (P1).

## 2. БАГ: краш при взаимодействии с магазином (приоритет: P0)

### 2.1 Симптом
При нажатии на элементы нижнего магазина (`ShopBar`) игра вылетает — либо
**мгновенно**, либо **через некоторое время** после нажатия (отложенный краш).

### 2.2 Что нужно сделать
- [ ] Воспроизвести краш и получить стек-трейс (Logcat) для обеих ситуаций:
  мгновенный вылет и отложенный вылет.
- [ ] Отложенный краш чаще всего означает гонку состояний (race condition) —
  проверить в первую очередь:
    - обращение к `state`/`viewModel` из `LaunchedEffect`/`coroutine`, запущенной
      до нажатия, которая стреляет уже после того как экран/данные изменились
      (например `delay()` в `PlanetButton`/`FloatingText`, который потом лезет
      в уже неактуальный список);
    - деление на ноль или индекс за пределами списка при пересчёте цены
      (`1.15.pow(count)`, `fleetItems[index]`, `planets.values.first()` если
      `planets` пуст);
    - обращение к `null` в `fleetItemsById[drone.type]` если для типа дрона
      нет конфига.
- [ ] Мгновенный краш — скорее всего NPE/IndexOutOfBounds прямо в обработчике
  клика конкретной кнопки магазина. Проверить каждый `onClick` в `ShopBar`,
  `ShopRow`, `PlanetRow` на обращение к данным, которых может не быть
  (например `viewModel.planets[id]` при `id`, которого больше нет в списке
  после того как поменялась редкость/состав планет — см. блок 3).

### 2.3 Критерий готовности
- Магазин открывается и закрывается многократно без падений.
- Покупка любого товара на всех трёх вкладках (Planets / Fleet / Click) не
  приводит к краху ни сразу, ни через 30–60 секунд ожидания после покупки.
- Добавлено логирование (или хотя бы try/catch с логом) вокруг обработчиков
  покупки, чтобы будущие краши было проще диагностировать.

## 3. Новая механика: редкость мусора и дронов (приоритет: P1)

### 3.1 Суть изменения
Сейчас дроны, судя по всему, приносят пассивный доход (DPS) просто по факту
владения ими. Это меняется:

> **Дроны больше не приносят деньги пассивно.** Деньги игрок получает
> **только** за счёт того, что дрон долетает и подбирает **предмет мусора**,
> который заранее появился на игровом поле. Ценность мусора зависит от его
> **редкости**. Дрон может подобрать только тот мусор, чья редкость **не выше**
> редкости, доступной этому дрону.

### 3.2 Редкость мусора (`ScavengeTarget`)

Вводим шкалу редкости (пять уровней — вынести в enum, значения ниже —
предложенные дефолты, баланс поправить на плейтесте):

| Редкость      | Вес спавна (шанс) | Множитель награды | Цвет метки (UI) |
|---------------|-------------------:|-------------------:|------------------|
| Обычный       | 60%                | x1                  | серый            |
| Необычный     | 25%                | x2.5                | зелёный          |
| Редкий        | 10%                | x6                  | синий            |
| Эпический     | 4%                 | x15                 | фиолетовый       |
| Легендарный   | 1%                 | x40                 | оранжевый        |

- Каждому заспавненному `ScavengeTarget` присваивается редкость случайно по
  весам из таблицы выше в момент спавна.
- Итоговая награда = `базовая_награда_мусора * множитель_редкости`
  (базовую награду завести отдельным параметром, чтобы её было удобно
  балансить без пересчёта таблицы множителей).
- В UI у объекта мусора должна быть визуальная метка редкости (цвет обводки/
  свечения — см. таблицу), чтобы игрок мог отличить редкий мусор от обычного
  до того, как дрон долетит.

### 3.3 Спавн мусора по таймеру

- Мусор (`ScavengeTarget`) появляется на поле не постоянно, а **по таймеру**:
  интервал между появлениями — случайное значение в диапазоне
  **от 2–3 секунд до 40 секунд** (точные границы — параметр, вынести в
  конфиг/константы, не хардкодить).
- Уточнить (требует решения дизайнера/разработчика, т.к. в исходном описании
  не указано однозначно):
    - спавнится **один** объект мусора за раз, или несколько параллельно;
    - если несколько — есть ли лимит одновременно существующих объектов на
      поле;
    - что происходит с заспавненным мусором, если его никто не подобрал за
      N секунд — исчезает или остаётся до подбора.
    - **Предположение по умолчанию** (если не уточнят иное): максимум 3
      объекта мусора на поле одновременно, каждый существует не дольше 60
      секунд, после чего исчезает и таймер спавна следующего запускается
      заново.

### 3.4 Редкость дронов и правило подбора

- Каждому типу дрона (`FleetConfig`, поле `id`, сейчас `d1..d9`) назначается
  **редкость дрона** — по той же шкале, что и у мусора (Обычный →
  Легендарный).
- Дрон **может подобрать мусор, чья редкость меньше либо равна** редкости
  самого дрона:

  ```
  можно_подобрать(дрон, мусор) = редкость(мусор) <= редкость(дрон)
  ```

  Пример: обычный дрон подбирает только обычный мусор. Дрон редкости
  «Редкий» может подобрать обычный, необычный и редкий мусор, но не
  эпический и не легендарный.

- Логика выбора цели дроном: дрон должен приоритетно лететь за **самым
  редким** доступным ему мусором из тех, что сейчас на поле (а не просто за
  ближайшим), чтобы более прокачанный флот ощутимо давал больше дохода.
  (Если это усложняет реализацию — как fallback допустимо лететь за
  ближайшим подходящим по редкости мусором; финальное решение — на
  усмотрение разработчика.)

### 3.5 Изменения в существующих сущностях

- `ScavengeTarget`: добавить поле `rarity` (enum).
- `FleetConfig` / `DroneData`: добавить поле `rarity` (enum) — редкость,
  которую дрон **может подбирать**.
- `GameViewModel.calculateDPS()` — убрать/пересчитать: пассивный DPS от
  владения дронами больше не начисляется. Вместо этого доход — событийный,
  начисляется в момент, когда дрон физически подбирает мусор (плюс
  всплывающий текст `+N`, аналогично клику по планете).
- Экран магазина (вкладка Fleet, `ShopRow`) — обновить описание товара: вместо
  `+X / sec` показывать что-то вроде `Собирает: Обычный, Необычный` (список
  редкостей, доступных этому дрону), чтобы игрок понимал, за что платит.

### 3.6 Критерий готовности
- В коде есть единый enum редкости, используемый и мусором, и дронами.
- Мусор спавнится по случайному таймеру в диапазоне 2–40 сек, с редкостью по
  весам из таблицы (веса — конфигурируемые, не хардкод в теле функции).
- Дроны не увеличивают `debris` просто по факту наличия в ангаре — доход
  идёт только за реально подобранный мусор.
- Дрон физически не может подобрать мусор выше своей редкости (проверено
  тестом: обычный дрон стоит рядом с легендарным мусором и не забирает его).
- В UI видно редкость мусора (цвет) и редкость/возможности дрона (в описании
  в магазине).
